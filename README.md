# ConvexHull

ConvexHull — проект, реализующий алгоритмы построения выпуклой оболочки множества точек на плоскости: алгоритм Джарвиса (Jarvis march) и алгоритм Грэхема (Graham scan).

Реализация выполнена на языке C++ с соблюдением стандарта C++17. Проект ориентирован как на образовательные цели (изучение алгоритмов вычислительной геометрии), так и на практическое применение в задачах обработки пространственных данных, компьютерной графики и геоинформационных систем.

---

## О проекте

Выпуклая оболочка — это минимальный выпуклый многоугольник, содержащий все точки заданного множества.

Проект демонстрирует:
- реализацию двух классических алгоритмов;
- сравнение их вычислительной сложности;
- модульное тестирование;
- поддержку мутационного тестирования;
- графическую визуализацию работы алгоритмов.

---

## Реализованные алгоритмы

### Алгоритм Джарвиса (Jarvis march)

Также известен как метод «заворачивания подарка» (Gift Wrapping).

Вычислительная сложность: O(nh),  
где n — количество точек,  
h — количество точек в выпуклой оболочке.

Особенности:
- простота реализации;
- эффективен при небольшом количестве точек;
- подходит для случаев, когда размер оболочки значительно меньше общего числа точек.

---

### Алгоритм Грэхема (Graham scan)

Вычислительная сложность: O(n log n).

Особенности:
- требует предварительной сортировки точек;
- эффективен для больших наборов данных;
- широко применяется в прикладных задачах вычислительной геометрии.

---

## Практическое применение

Алгоритмы построения выпуклой оболочки используются в:

- обработке данных лазерного сканирования (LiDAR);
- построении зон видимости в навигационных системах;
- выделении границ объектов в задачах компьютерного зрения;
- оптимизации маршрутов и анализе пространственных данных в ГИС;
- построении минимальных ограничивающих многоугольников;
- задачах триангуляции и анализа формы объектов.

## Архитектура проекта

Проект построен с использованием CMake и поддерживает различные конфигурации сборки.

### Структура каталогов

    app/         — графическое приложение (SFML)
    libraries/   — внешние зависимости (Google Test, SFML)
    tests/       — модульные тесты
    Dockerfile   — конфигурация Docker для мутационного тестирования

---

## Варианты сборки

Сборка управляется флагами CMake.

| Флаг         | Назначение                                                   | Платформа |
|--------------|--------------------------------------------------------------|------------|
| BUILD_APP    | Сборка графического приложения с использованием SFML       | Windows    |
| BUILD_TESTS  | Сборка и запуск модульных тестов (Google Test)             | Windows    |
| BUILD_MULL   | Сборка для мутационного тестирования с использованием Mull | Linux      |

Ограничения конфигураций:

- При включении BUILD_MULL флаги BUILD_APP и BUILD_TESTS автоматически отключаются.
- На платформе Windows флаг BUILD_MULL игнорируется.
- В Linux при активном BUILD_MULL используется компилятор clang-15 и подключается плагин Mull (путь задаётся переменной MULL_PLUGIN).

Такая организация исключает некорректные комбинации флагов и предотвращает сборку неподдерживаемых конфигураций.

## Мутационное тестирование (Mull + Docker)

Для упрощения работы с Mull подготовлен Docker-образ, содержащий:

- Clang 15
- Mull 15
- CMake
- необходимые зависимости

### Сборка Docker-образа

    docker build -t mull .

### Запуск контейнера

    docker run -it --rm -v ${PWD}:/project mull

После запуска контейнера вы окажетесь внутри среды с примонтированной директорией проекта.

Далее можно выполнить сборку с флагом:

    -DBUILD_MULL=ON

и запустить мутационное тестирование:

    mull-runner-15 ./ConvexHull_mull

## Графическое приложение

Приложение использует библиотеку SFML для визуализации точек и построенной выпуклой оболочки.

### Входные данные

Программа ожидает файл points.txt.

Файл должен находиться в той же директории, что и исполняемый файл.

### Формат файла

- каждая строка содержит два целых числа — координаты X и Y;
- числа разделяются пробелом;
- пустые строки игнорируются.

### Пример содержимого

    10 100
    100 100
    200 200
    150 300
    50 250

### Ограничения

- допустимый диапазон координат: от 0 до 600;
- диапазон соответствует отображаемой координатной сетке приложения.
